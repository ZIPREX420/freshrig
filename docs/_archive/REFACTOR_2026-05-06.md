## Pass 2 — bundle splitting + UX hardening (2026-05-06, post-self-audit)

After the first pass shipped with several real flaws (caught by self-critique),
the following corrective + extension changes landed:

### New helpers
- `src/lib/lazy.ts` — `lazyNamed<M, K>()` (type-preserving lazy wrapper) +
  `preloadModule()` (hover/idle prefetch primitive).
- `src/lib/index.ts` — barrel export for `runtime`, `invoke`, `lazy`. Callers
  now write `import { isTauri, invokeOrToast, lazyNamed } from "../../lib"`.
- `src/components/ui/DelayedFallback.tsx` — defers Suspense fallback rendering
  by 120 ms so chunks that resolve quickly never paint the spinner.

### Vite config — chunkFor predicate hardened
- Each package matcher now uses bracketed `/foo/` paths to prevent substring
  collisions (e.g. `framer-motion` matching `framer-motion-utils`).
- React-core fallback moved last; specific React-* packages
  (`react-markdown`, `react-error-boundary`, `react-hotkeys-hook`) all named.
- framer-motion family expanded to all four variants
  (`framer-motion`, `motion`, `motion-dom`, `motion-utils`).
- Added explicit caveat in code comments: `manualChunks` cannot evict
  modules from the entry chunk if reachable through eager imports — sets
  realistic expectations for the user reading dist/assets/.
- `modulePreload.polyfill: false` (Tauri ships modern WebViews; polyfill
  is dead weight).

### CSP — dev-mode override
- `tauri.conf.json` gained a `devCsp` field with HMR-friendly relaxations
  (Vite's `'unsafe-eval'` for source maps, the localhost dev server origin,
  HMR ws). Production `csp` unchanged (still strict).

### App.tsx rewrite
- All 18 lazy calls collapsed from 4-line `.then(m => ...)` boilerplate
  to single-line `lazyNamed(() => import(...), "ComponentName")`.
- **OnboardingWizard, CommandPalette, ShortcutHelp moved BACK to eager
  imports** — these are first-launch / power-user paths where lazy-load
  flicker is hostile UX.
- WhatsNewModal stays lazy (drags react-markdown), but its show effect now
  calls `preloadModule(...)` concurrently with `setShowWhatsNew(true)` so
  the chunk is in cache by render.
- `<Suspense>` moved OUTSIDE `<AnimatePresence>` so the route transition
  wraps the resolved component, not the spinner — eliminates the
  "Old → Spinner → New" three-state flicker.
- Performance instrumentation: `performance.mark("freshrig:app-module-evaluated")`
  at module top, `performance.mark("freshrig:app-mounted")` + `measure()` in
  the first effect. Surfaces in DevTools → Performance → User timing.
- `RouteFallback` replaced by `DelayedFallback` with 120 ms grace.
- `onCloseRequested` listener now uses the `cancelled` flag pattern to
  prevent late-resolve race on unmount.

### Sidebar.tsx — preload on hover
- New `ROUTE_PRELOADERS` map (15 entries, one per lazy route) with
  `onMouseEnter` / `onFocus` / `onTouchStart` triggers. Hover or
  keyboard-focus fires the dynamic import; click finds the module already
  in the cache. Errors are swallowed (the eventual click load surfaces them).

### TitleBar.tsx — onResized cleanup race fixed
- Replaced `unlistenP.then((fn) => fn())` cleanup with
  `let cancelled = false; let unlisten; ...; if (cancelled) fn(); else unlisten = fn;`
  pattern. Prevents the listener from leaking when the component unmounts
  before `onResized` resolves.
- Migrated to `import { isTauri } from "../../lib"` (barrel).

### Why these matter
- The first pass's `manualChunks` predicate had three real bugs (substring
  collision, `react-*` fallback ordering, missing `motion` package).
- The first pass made first-launch worse (lazy-loading OnboardingWizard).
- The first pass introduced a 3-state route transition flicker.
- The first pass left `cargo fmt --check` failing on lib.rs by writing
  single-line assertions rustfmt wanted multi-line — re-format applied.
- The CSP rejected by Tauri schema (`_csp_note`) was the symptom of writing
  a docs string into a strict-schema config — moved to REFACTOR notes,
  CSP itself kept.

## Validation checklist (run locally)

This pass cannot be validated from inside the agent session. Before committing:

```sh
# 1. Type-check
npx tsc --noEmit

# 2. Rust lint + format
cargo clippy --manifest-path src-tauri/Cargo.toml -- -D warnings
cargo fmt --manifest-path src-tauri/Cargo.toml -- --check

# 3. Tests
cargo test --manifest-path src-tauri/Cargo.toml
# ↑ should now show 8 new tests passing (7 scrub + 1 release_gate no-op)

# 4. Audit (the new CI step)
cargo install --locked cargo-audit
cargo audit --file src-tauri/Cargo.lock

# 5. Frontend build
npm run build

# 6. Smoke test the dev app (verify CSP doesn't break the UI)
npm run tauri dev
```

If `cargo audit` flags anything new, add a documented entry to
`src-tauri/.cargo/audit.toml` or upgrade the offending crate.

If `npm run tauri dev` shows CSP violations in DevTools, the CSP needs
broadening — but err on the side of *narrowing*, not loosening, by
extracting the offending inline script/style into an external file.
